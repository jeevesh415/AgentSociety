# Memory (Finalize)

Finalize phase of the pipeline — batch-persist internal states generated by the cognition skill during the current step (need adjustments, emotions, thoughts) to long-term memory.

## Why finalize

The `cognition` skill (dynamic, priority=40) writes multiple records to `agent._cognition_memory` during execution. Memory must flush after all dynamic skills complete, otherwise data will be lost.

## What It Does

1. **Flush cognition memory** — Batch-commit entries from `_cognition_memory` grouped by type to long-term memory (pure memory operation, 0 LLM calls).
2. **Intention query** (optional) — Query current intention every 2 steps (1 LLM call), triggered only when cognition skill has run.

## Key Methods on Agent

| Method | Purpose |
|--------|---------|
| `_flush_cognition_memory_to_memory()` | Batch-commit cognition scratchpad to long-term |
| `_query_current_intention()` | LLM query for current intention (optional) |
